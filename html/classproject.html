Kitty Combos
Made by Human Systems
The Goal
The goal of this project was to create a cat-themed Candy-Crush/Bejeweled genre game. This purpose of this game is to increase the players score by matching 3 or more adjacent tiles of cat pictures. The player will be given a set number of turns upon starting the game (likely 10-15), with the goal of earning the highest score possible.
The Plan
below are a couple of diagrams used by Human Systems to plan out core components of Kitty Combos prior to coding.
The class diagram is used to display what objects in the project will do and how those objects relate to each other. Kitty Combos makes use of these 4 class models within the game.
By connecting use cases within the game, the activity diagram was used to describe the flow of the game and was referenced many times during development. This diagram is great for providing guidance when planning out functionality of the games methods.
My Contribution
My main responsibilities in the project were to create the server to server our game and to create a database/file system to read data to and write data from.

Server

Kitty Combos needed a server in order to retrieve data from the CATAAS API and to store and read data on a file that stores data to be displed on the games leaderboard. One of the more challenging parts of creating a server for this app was understanding how to host static files and why they needed to be hosted.
